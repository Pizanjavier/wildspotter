# Legal & Restriction Data Rules

Full pipeline reference: `docs/legal-monitoring-pipeline.md`

## Two Subsystems

### 1. Static Per-Spot Checks (Pipeline Stage)
- `workers/pipeline/legal.py` runs `ST_Intersects` against MITECO PostGIS tables (~1ms/spot)
- Checks: Natura 2000, National Parks, Coastal Law (DPMT + Servidumbre + Terrenos)
- Catastro uses live REST API `Consulta_RCCOOR` (~300ms/spot)
- Results stored in `legal_status` JSONB column with `blocked: true/false`
- No spots rejected — legal data is informational, users filter at query time

### 2. Dynamic Legal Monitoring Pipeline
- 89 sources: BOE, AEMET, 8 RSS, 8 HTML, 47 BOPs, 6 email (n8n schedule → worker IMAP poll), 1 baseline
- `workers/legal/scheduler.py` runs as `legal-watcher` Docker service
- Change detection: SHA-256 hash comparison (`workers/legal/source_monitor.py`)
- Classification: keyword gate (regex, free) → LLM (Ollama dev / Haiku prod) via `workers/legal/classifier.py`
- Deduplication: exact hash + fuzzy pg_trgm + authority ranking (`workers/legal/dedup.py`)
- Health tracking: GREEN → YELLOW (3 failures) → RED (7 failures), ntfy.sh alerts

## Data Sources (Spain)
- **MITECO shapefiles** — PostGIS: Red Natura 2000 (SRID 3042), National Parks (SRID 25830), Coastal Law (SRID 25830)
- **Catastro REST API** — Land classification via `Consulta_RCCOOR` (~300ms/query)
- **BOE XML API** — National gazette (6h poll)
- **AEMET REST API** — Fire risk (requires `AEMET_API_KEY`, 6h poll)
- **17 CCAA gazettes** — RSS (8), HTML (8), email/n8n (4), with Catalan/Galician keyword variants
- **47 provincial BOPs** — HTML (33), PDF (10), Playwright (7)
- **Baseline CCAA** — 18 verified tourism decree articles covering all 17 CCAAs (`003_legal_baseline.sql`)
- **Baseline Parks** — 18 PRUG overnight rules for parks overlapping spots, geometry-linked (`004_park_rules_baseline.sql`)
- **OSM boundary** — Spain admin boundary for coastline clipping of legal zone tiles

## Database Tables
- `legal_source_state` — 89 monitored sources with hash-based change detection
- `legal_documents` — parsed restrictions with geometry, confidence tiers, `decree_articles` JSONB
- `municipalities` — 8,132 Spanish municipalities with INE codes + PostGIS boundaries
- `priority_municipalities` — 63 high-enforcement municipalities for backfill tracking

## CCAA Baseline Decree Articles
Stored as `DecreeArticle[]` array in `decree_articles` JSONB:
```typescript
{ number: string; title: string; text_verbatim: string;
  legal_distinction: string | null; max_stay_hours: number | null;
  restrictions: string[]; exceptions: string[] }
```
Key rules: Baleares=prohibited(0h), Extremadura=permitted(72h), Aragón/Galicia/Navarra=tolerated(48h), Cataluña=tolerated(24h)

## API Endpoints
- `GET /legal/documents?lat=X&lon=Y&radius=Z` — location-based documents (Fastify)
- `GET /legal/documents/:id` — single document (Fastify)
- `GET /legal/sources` — monitored sources + health (Fastify)
- `GET /legal/decrees/:ccaa` — CCAA tourism decrees (Fastify)
- `GET /legal/tiles/:z/:x/:y.pbf` — pre-generated MVT tiles z4-z12 (Fastify, static files)
- Worker internal: `/legal/health`, `/legal/ingest-email`, `/run/legal-poll`

## Legal Zones Map Overlay (Vector Tiles)
- Pre-generated MVT tiles at z4-z12 in `data/legal-tiles/{z}/{x}/{y}.pbf`
- ~1,038 non-empty tiles (~74 MB), zero PostGIS cost at runtime
- Generated by `workers/generate_legal_tiles.py` (materialized view, ST_Subdivide, coastline-clipped)
- DPMT boundary as 20m buffer polygon matching `legal.py`'s `ST_DWithin(geom, point, 20)`
- MapLibre: semi-transparent red fill (`#EF4444`, 15% opacity) with red border
- Toggled by `showLegalZones` in Config screen
- Regenerate: `docker-compose exec worker python generate_legal_tiles.py`

## Frontend — Unified Legal Situation
- `LegalSituation.tsx` — unified component replacing LegalChecklist + LegalDocumentsList in spot detail
  - Collapsed by default: shows overnight verdict card ("Pernocta restringida/tolerada/permitida")
  - Expands to show merged static checks + dynamic documents sorted by severity
  - Verdict priority: fire ban → private land → National Park → Coastal → camping ban → Natura 2000 → CCAA decree → allowed
  - National Parks override CCAA decree tolerance (park regs are stricter)
- `src/utils/legal-verdict.ts` — shared `getOvernightLevel()` for map dots + spot card borders
  - 3 levels: `allowed` (white/no border), `restricted` (amber `#FBBF24`), `prohibited` (red `#EF4444`)
- `DecreeArticleCard.tsx` — decree article renderer (used inside LegalSituation)
- `ConfidenceTierBadge.tsx` — pill badge (used inside LegalSituation)
- `LegalDisclaimer.tsx` — "Solo informativo" disclaimer
- `LegalChecklist.tsx` / `LegalDocumentsList.tsx` — legacy, exported but not rendered

## Important Notes
- Environmental boundaries are legally binding — err on the side of caution
- Coastal Law (Ley de Costas) has complex buffer zones — don't simplify
- Response schemas for JSONB fields must include `additionalProperties: true` to prevent Fastify stripping
- `LegacyCipherAdapter` (SECLEVEL=0 + TLSv1) needed for legacy BOP servers
- Keyword pattern includes Catalan/Galician variants for bilingual CCAAs
- Location query filters: `LENGTH(title) > 40`, `restriction_type != 'other'`, `needs_review = FALSE`
