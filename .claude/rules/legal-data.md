# Legal & Restriction Data Rules

## Data Sources (Spain)
- **MITECO shapefiles** — Imported into PostGIS: Red Natura 2000 (SRID 3042), National Parks (SRID 25830), Coastal Law zones (SRID 25830)
- **Catastro REST API** — Land classification via `Consulta_RCCOOR` (~300ms/query)
- **OSM boundary** — Spain admin boundary used for coastline clipping of legal zone tiles

## Per-Spot Legal Checks (Pipeline)
- `workers/legal.py` runs `ST_Intersects` queries against MITECO PostGIS tables (~1ms/spot)
- Catastro uses a live REST API call per spot
- Results stored in `legal_status` JSONB column with `blocked: true/false`
- No spots are rejected — legal data is informational, users filter at query time

## Legal Zones Map Overlay (Vector Tiles)
- Pre-generated MVT tiles at z4–z10 stored in `data/legal-tiles/{z}/{x}/{y}.pbf`
- Served as static files by `GET /legal/tiles/:z/:x/:y.pbf` — zero PostGIS cost at runtime
- Generated by `workers/generate_legal_tiles.py` which:
  - Creates materialized view `legal_zones_3857` (pre-transformed to EPSG:3857, subdivided with `ST_Subdivide(geom, 256)`)
  - Includes DPMT boundary line as 20m buffer polygon (matches `legal.py`'s `ST_DWithin(geom, point, 20)` check)
  - Clips to Spain's OSM administrative boundary to exclude marine zones
  - Writes ~1,038 non-empty tiles (~74 MB total)
- Regenerate when MITECO shapefiles are updated: `docker-compose exec worker python generate_legal_tiles.py`
- MapLibre renders as semi-transparent red fill (`#EF4444`, 15% opacity) with red border
- Toggled by user setting `showLegalZones` in Config screen

## Legal Status Display
- Spot detail shows checklist-style status per layer (green check / red X)
- Map overlay shows all restriction areas as red zones when enabled
- Source attribution: «© Ministerio para la Transición Ecológica y el Reto Demográfico»

## Important Notes
- Environmental boundaries are legally binding — err on the side of caution
- Coastal Law (Ley de Costas) has complex buffer zones — don't simplify
- Legal results are stored in the `legal_status` JSONB column in PostGIS
- Response schemas for JSONB fields must include `additionalProperties: true` to prevent Fastify from stripping nested properties
